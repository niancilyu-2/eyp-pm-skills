# Roadmap recommendation template (transparent RICE + Now/Next/Later)

The point is a recommendation a PM can **defend line by line**. Show the inputs and the math; never
present a score as if it fell from the sky.

## Recommendation (one paragraph)
> We recommend **[initiative]** for **[Now / Next / Later]**, because customers get
> **[direct customer value, in the customer's own terms — hours saved, risk removed, outcome
> enabled]**, which shows up commercially as **[$ARR retained/expanded, reach]**. This displaces /
> defers **[what moves]**.

## Transparent RICE
For each factor, show the sourced input AND the rationale. Score, then compute.

| Factor | Sourced input | Value | Rationale |
|--------|---------------|-------|-----------|
| **Reach** | e.g. accounts/users affected per quarter `[churn-data.xlsx]` | _n_ | … |
| **Impact** | direct customer value (their units: hours, risk, outcomes) `[source]` + business value ($ARR at risk/upside) `[source]` | 0.25–3 | both halves stated; customer value leads |
| **Confidence** | strength of evidence (triangulated? sample size?) | % per rigor mapping (High 85–100 / Med 60–80 / Low <60) | **Weak/single-source signals cap at Low (≤60%)**; strong multi-source triangulation earns High — differentiate, don't default |
| **Effort** | rough person-months (refine in pm-plan-and-estimate) | _n_ | … |

**RICE = (Reach × Impact × Confidence) / Effort = [show the arithmetic]**

> Note: Effort here is a placeholder for prioritization. The real, codebase-grounded estimate comes
> from the `pm-plan-and-estimate` skill.
