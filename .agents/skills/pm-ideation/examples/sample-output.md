# EXAMPLE OUTPUT — pm-ideation (excerpts)

The parts a finished deliverable should contain. **Structural only — never copy figures,
citations, or conclusions from examples.**

## Exec summary (shape)
The dominant pattern named in one sentence — as a failing job-to-be-done, not a feature label —
with the deduped cluster size and dollars next to it, how it compares to the next-largest reason,
a Now/Next/Later recommendation, and a confidence figure with its basis.

## What makes it defensible (the parts to imitate)
- **Clustering shown:** sub-themes explicitly merged into one theme, with per-sub-theme counts.
- **Dedup shown:** duplicate CRM rows named and excluded, math visible.
- **Per-leg evidence table:** each independent source with its count toward the cluster.
- **Traps dodged, in writing:** the already-planned ask and the different-buying-center account
  are examined and set aside, with citations for why.
- **A red-team that can change the answer:** the strongest objection is stated, and if it
  partially wins, the recommendation visibly changes shape.
- **Success criteria with a guardrail**, and [Assumption] labels on unmeasured baselines.
- **Evidence appendix:** every figure with a file + row/slide/thread locator.
